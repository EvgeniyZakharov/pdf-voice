# PDF Voice — план разработки

iOS-приложение для чтения PDF вслух. Полностью бесплатный нативный стек, офлайн, приоритет — русский язык.

Легенда статусов: ✅ сделано · 🔜 следующее · ⏳ запланировано · 💡 бэклог.

> **Примечание.** Это исходный MVP-план. Актуальная архитектура и текущее
> состояние — в `CLAUDE.md` (что сделано/бэклог) и `ARCHITECTURE.md` (целевая
> архитектура). Все вехи M1–M6 завершены; добавлены нейросетевой Silero TTS на
> своём сервере и подготовка к App Store.

## 1. Цель и позиционирование

**Проблема рынка:** бесплатные читалки PDF вслух плохи — (1) не читают сканы, (2) читают «мусор» (колонтитулы, номера страниц, переносы, оглавления), (3) нет синхронной подсветки и навигации.

**Наш ответ:** офлайн, бесплатно, без лимитов на символы. Качество вытягиваем UX-ом: чистка текста + синхронная подсветка + удобная навигация + сохранение позиции.

**Не-цели MVP:** аккаунты, облако/синхронизация между устройствами, облачный TTS, Android.

## 2. Технологический стек

| Слой | Технология | Статус |
|---|---|---|
| UI | SwiftUI | ✅ |
| Открытие/рендер PDF | PDFKit (`PDFView`, `PDFDocument`, `PDFPage`) | ✅ |
| Озвучка | AVFoundation (`AVSpeechSynthesizer`) | ✅ |
| Разбиение на предложения | NaturalLanguage (`NLTokenizer`) | ✅ |
| Хранилище | Codable JSON (под min iOS 16; не SwiftData) | ✅ |
| OCR (сканы) | Vision (`VNRecognizeTextRequest`) | ✅ |
| Фон/Lock screen | `AVAudioSession` + `MPNowPlayingInfoCenter` + `MPRemoteCommandCenter` | ✅ |
| Нейросетевой TTS | Silero (свой сервер `tts.pdf-voice.com`, офлайн-фолбэк на системный) | ✅ |

Min iOS **16.0**. Сборка: XcodeGen (`project.yml` → `xcodegen generate`), `.xcodeproj` в репозитории не хранится. Bundle id `com.pdfvoice.app`.

## 3. Архитектура (текущая)

```
PDFVoice/
├─ App/PDFVoiceApp.swift           ✅ точка входа
├─ Models/LibraryItem.swift        ✅ модель библиотеки (Codable)
├─ Library/
│  ├─ DocumentStore.swift          ✅ импорт PDF + JSON-хранилище + прогресс
│  └─ LibraryView.swift            ✅ список, импорт (Files), удаление
├─ Reader/
│  ├─ PDFKitView.swift             ✅ PDFView + подсветка + скролл + тап-хиттест
│  ├─ ReaderViewModel.swift        ✅ загрузка, детект сканов, проброс прогресса
│  └─ ReaderView.swift             ✅ экран чтения + плеер + кнопка «Отсюда» + меню скоростей/голоса
└─ Core/
   ├─ PDFTextExtractor.swift       ✅ текст → предложения, маппинг диапазонов
   ├─ TextNormalizer.swift         ✅ чистка: колонтитулы/номера/лидеры/переносы/аббревиатуры
   ├─ TTSProvider.swift            ✅ абстракция движка (шов под облачный TTS)
   └─ SpeechEngine.swift           ✅ AVSpeechSynthesizer, скорость, голоса
```

`SpeechEngine` за протоколом `TTSProvider` — готовый шов, чтобы позже подключить облачный TTS как премиум без переписывания Reader.

## 4. Вехи

### ✅ M1 — Каркас + чтение текстового PDF
Импорт PDF из Files, библиотека с персистентностью, открытие в `PDFView`, извлечение текстового слоя, базовая озвучка Milena, play/pause/скорость, сохранение/восстановление позиции, детект сканов.

### ✅ M2 — UX чтения
- Синхронная подсветка предложения на странице (`PDFSelection` + `highlightedSelections`), привязка к делегату `didStart` — выделяется ровно звучащее предложение.
- Плавная авто-прокрутка к текущему предложению (только при смене, без дёрганья).
- **Тап по предложению → всплывающая кнопка «▶ Отсюда»** (хит-тест по `selectionsByLine`).
- **Дискретные скорости** 0.5×…2.5× с мгновенным применением (перезапуск текущего предложения).
- Меню выбора русского голоса (Enhanced/Premium).

### ✅ M4 — Чистка текста (`TextNormalizer`)
- Колонтитулы (повтор по страницам), номера страниц, строки-лидеры оглавления (5+ точек).
- Склейка переносов по дефису, схлопывание пробелов и многоточий.
- Раскрытие безопасных аббревиатур («т.е.», «т.к.», «№»…).
- Выброшенные строки не ломают подсветку (сохранение карты смещений UTF-16).

### ✅ Навигация по страницам
- Скраббер страниц (слайдер + счётчик «N/Total») — быстрый прыжок на большие расстояния.
- Сетка миниатюр всех страниц (постраничный листинг) с переходом по тапу.
- Двусторонняя синхронизация: бегунок едет при чтении, книга прыгает при перетаскивании.
- Непрерывная вертикальная прокрутка; кнопки плеера ◀▶ — по предложениям.

### ✅ M3 — Сканы (OCR)
- Детект сканов → Vision OCR (`VNRecognizeTextRequest`, `ru-RU`/`en-US`, accurate).
- Асинхронная обработка с индикатором прогресса по страницам.
- Подсветка по OCR-боксам через PDF-аннотации (`PDFSelection` тут неприменим); тап «Отсюда» тоже работает по боксам.

### ✅ M5 — Фон и системная интеграция
- `AVAudioSession .playback` + Background Mode `audio`.
- `MPNowPlayingInfoCenter` (название/статус) + `MPRemoteCommandCenter` (play/pause/след./пред. предложение с Lock screen и наушников).
- Корректное снятие Now Playing и команд при выходе из читалки (`endSession`).

### ✅ M6 — Полировка
- Закладки, прогресс-бар по документу, переход к главе/оглавлению.
- Тёмная тема, иконка приложения, онбординг.
- Настройки: пауза между предложениями, шаг перемотки, голос по умолчанию.
- Таймер сна, обработка ошибок/повреждённых PDF.

## 5. Бэклог / монетизация (после MVP)

- ✅ **Нейросетевой TTS** — реализован через Silero на своём сервере (бесплатно, русские голоса с ударениями). Облачный TTS (ElevenLabs/OpenAI) остаётся опцией для премиум-качества.
- ✅ **Числа → слова** и **ударения-омографы** — сделаны (Фаза 1). Остаётся: контекстные омографы (за́мок/замо́к) и род числительных в именительном — см. бэклог `CLAUDE.md`.
- 💡 **Листинги кода** — сейчас читаются как мусор; нужен аккуратный детект, чтобы не выкинуть реальный текст.
- 💡 **Мусор титульных страниц** — нужен layout-анализ порядка текста.
- 💡 Экспорт в аудиофайл, iCloud-синхронизация позиции, режим reflow («только текст»).
