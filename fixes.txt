Здесь будут описаны все планируемые изменения, замечания и баги.

1. Списки не читает номера. Должен читать номер.  [ГОТОВО: expandListMarker -> "первое,"]
2. Игнорируются числа, это неправильно  [ГОТОВО: expandNumbers, рус. числительные]
3. Не работает скорость чтения на языках silero  [ГОТОВО: AVAudioPlayer.enableRate+rate]
4. Скорость сделать 0.5, 0.75, 1.0, 1.25, 1.5, 2.0  [ГОТОВО: speedOptions]

A. ЧИСТКА — что НЕ читать
  [есть] номера страниц (isPageNumberLine)
  [есть] колонтитулы / повторяющиеся строки (detectBoilerplate)
  [есть] строки-лидеры оглавления "......" (isLeaderLine)
  [есть] переносы по дефису "сло-\nво" -> "слово"
  [есть] схлопывание пробелов и пустых строк
  [есть] ссылки и email (NSDataDetector)
  [есть] многоточия / лидеры из точек
  [ГОТОВО] копирайт по шаблону: ©, "all rights reserved", "все права защищены",
         дата-штампы, "Page N"/"Стр. N", "N of N"/"N из M" (TextNormalizer.isJunkLine)
  [НЕТ] сноски и ссылки на литературу: "[12]", "(Иванов, 2020)", "ibid.",
         номера сносок в конце строки
  [НЕТ] подписи к рисункам/таблицам: "Рис. 3", "Таблица 1", "Fig. 2"
         (сейчас просто отдаём тишину, если в строке нет букв)

B. НОРМАЛИЗАЦИЯ — что читать ПРАВИЛЬНО  <- сюда относятся п.1 и п.2
  [есть] аббревиатуры: т.е., т.к., см., № ...
  [ГОТОВО] ЧИСЛА СЛОВАМИ: "2024" -> "две тысячи двадцать четыре" (integerToWords,
         группы/разряды, род для тысяч). >9 цифр читаются почифренно (телефоны).
         Проценты: "50%" -> "пятьдесят процентов". Десятичные: "3,14" -> "три
         запятая четырнадцать". Числа теперь раскрываются на клиенте ДО Silero,
         поэтому серверная "тишина на без-буквенном тексте" их больше не глушит.
  [ГОТОВО] НУМЕРАЦИЯ СПИСКОВ: "1." / "1)" в начале пункта -> "первое," (expandListMarker)
  [ГОТОВО] номера разделов "2.1 Название" -> "два точка один Название" (expandNumbers)
  [бэклог] ударения-омографы (за́мок / замо́к)

C. СЕГМЕНТАЦИЯ
  [есть] разбивка на предложения (NLTokenizer, кириллица)
  [есть] склейка предложения, разорванного границей страницы (mergeCrossPage)
  [НЕТ] лимит длины фрагмента для TTS: чанк <= ~2500 символов, резать по
         границам предложений/абзацев (защита от очень длинных предложений)
  [НЕТ] разрыв по абзацам как отдельная единица паузы

D. СТРУКТУРА КНИГИ
  [ГОТОВО] детект глав/заголовков: "Глава N", "CHAPTER N", "# Заголовок",
         "1.2 Название" + спец-разделы (TextNormalizer.isHeadingText)
  [ГОТОВО] спец-разделы: Пролог, Эпилог, Предисловие, Введение, Заключение...
  [ГОТОВО] заголовок = отдельная единица, не склеивается с абзацем (mergeCrossPage)
  [НЕТ] (опц.) голосовое объявление главы перед её текстом

E. ТЕМП И ПАУЗЫ (ACX / комфорт восприятия)
  [есть] пауза между предложениями (pauseBetweenSentences = 0.3с)
  [ГОТОВО] доп. пауза после заголовка (+0.7с, Sentence.isHeading -> headingPause)
  [частично] иерархия пауз: ещё нет отдельной паузы между абзацами
  [реком.] базовая скорость чуть ниже обычной для длинного чтения

--------------------------------------------------------------------
ПРИОРИТЕТ ВНЕДРЕНИЯ (предложение)
  1) Числа словами (рус.) + чтение номеров списков   <- закрывает п.1, п.2
  2) Откатить "тишину на числах" после п.1 (числа должны звучать)
  3) Фильтр сносок / ссылок на литературу / подписей к рис.
  4) Иерархия пауз (абзац / глава) + детект заголовков
  5) Лимит длины TTS-фрагмента
