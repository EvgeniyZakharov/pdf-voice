# Корректная обработка форматов

Каждый формат имеет свою природу. Главная ошибка — единая логика «для всех файлов».
Базовое деление: **fixed-layout** (вёрстка зашита — PDF, сканы) vs **reflow** (текст
перетекает под экран — EPUB, FB2, TXT, DOCX).

<a name="деление"></a>
## 1. Fixed-layout vs reflow

| | Fixed-layout (PDF/сканы) | Reflow (EPUB/FB2/TXT/DOCX) |
|---|---|---|
| Единица позиции | страница | символьное смещение / % / глава |
| Прогресс | «стр. N из M» | «% книги», «глава N/M» |
| Шрифт/размер | зашит в вёрстку | пользовательский, Dynamic Type |
| Темы чтения | фильтр/яркость | полноценные (свой фон+текст) |
| Подсветка озвучки | бокс на странице (координаты) | диапазон символов в тексте |
| Главы | часто только из outline | из структуры документа |

Из этого следует: **модель прогресса, навигации и подсветки выбирается по типу**, а не
одинаково. PDF Voice держит fixed-layout через PDFKit, reflow — через единый плоский текст
с символьными смещениями.

<a name="pdf"></a>
## 2. PDF

PDF — fixed-layout. Бывает двух сильно разных видов, и это надо определять **постранично**:
- **С текстовым слоем** — извлекаем текст напрямую.
- **Скан (картинки)** — текстового слоя нет → нужен **OCR**.

**Ловушки PDF:**
- **Битая кодировка (CMap).** Текстовый слой есть, но `extractText` даёт мусор/нечитаемое.
  Нельзя判ить «есть ли текст» по простому наличию символов — нужен порог плотности
  настоящих букв; ниже порога → OCR. (В PDF Voice: порог ~0.35.)
- **Смешанный документ.** Часть страниц текстовые, часть — сканы. Классифицировать
  постранично и обрабатывать соответственно; порядок предложений = порядок страниц.
- **Колонтитулы и номера страниц** попадают в озвучку как мусор — детектировать и убирать
  (повторяющиеся строки по окну страниц), не ломая заголовки глав.
- **Пустые страницы** — детектировать (дисперсия яркости рендера), не тратить на них OCR.
- **Переносы слов** на конце строки/страницы — склеивать; межстраничные предложения — мерджить.
- **Многоколоночная вёрстка / таблицы / формулы** — порядок чтения нетривиален; как минимум
  не выдавать кашу, в идеале — анализ layout.

**OCR:** язык(и) под аудиторию (ru-RU/en-US), off-main, прогрессивно (показывать прогресс),
кэшировать результат; для подсветки — суб-строчные боксы (координаты слов), а не вся строка.

<a name="epub"></a>
## 3. EPUB

EPUB — основной reflow-формат, по сути zip с XHTML. Обрабатывать как структуру, а не как
один текст.

**Что распарсить:**
- **`container.xml`** (`META-INF/container.xml`) → путь к OPF (`.opf`).
- **OPF manifest + spine** — spine задаёт **порядок чтения** документов (XHTML). Читать
  строго по spine, а не по алфавиту файлов.
- **Навигация/оглавление:** EPUB 3 — `nav.xhtml` (`epub:type="toc"`); EPUB 2 — `toc.ncx`
  (`navMap`). Брать заголовки и якоря глав.
- **Метаданные** (Dublin Core в OPF): `dc:title`, `dc:creator` (автор), `dc:language`.
- **Обложка:** `manifest` item с `properties="cover-image"` (EPUB3) или `meta name="cover"`
  (EPUB2).

**Извлечение текста из XHTML:** убрать разметку, сохранив абзацы/заголовки (для пауз и
определения глав); обрабатывать сущности (`&nbsp;` и т.п.); порядок — по spine.

**Ловушки EPUB:**
- **Сноски/примечания** — не вставлять inline в поток озвучки как мусор; в идеале выносить.
- **Изображения** в reflow-озвучке пропускаются (или alt-текст), но не должны ломать поток.
- **Кодировка** — обычно UTF-8, но проверять declared encoding.
- **DRM** — зашифрованный EPUB не парсится; сообщить понятно, не падать.
- **Вложенное/глубокое оглавление** — поддержать иерархию (части/главы) хотя бы плоско.

<a name="fb2"></a>
## 4. FB2

FB2 — один XML-файл, популярен в рунете (важно для русскоязычной аудитории).

**Структура:**
- **`<description>`** → `<title-info>`: `<book-title>`, `<author>` (first/last-name),
  `<lang>`, обложка в `<coverpage>` (ссылка на `<binary>`).
- **`<body>`** → `<section>` (вложенные) — это **главы**; `<title>` секции — заголовок главы.
- **`<binary>`** — встроенные картинки (base64), в т.ч. обложка.

**Обработка:** идти по секциям рекурсивно → главы; `<p>` → абзацы; `<emphasis>/<strong>` —
снять, сохранив текст; `<epigraph>/<poem>/<cite>` — текст с разумными паузами. Кодировка —
из XML-декларации (часто windows-1251 в старых файлах!) — обязательно уважать, иначе
кириллица превратится в мусор.

**Ловушки FB2:** windows-1251 vs UTF-8; вложенные секции (иерархия глав); сноски (`<a
type="note">` + `<body name="notes">`).

<a name="docx"></a>
## 5. DOCX

DOCX — zip с OOXML. Reflow.

**Что распарсить:**
- **`word/document.xml`** — основной текст: `<w:p>` (абзацы), `<w:r><w:t>` (прогоны текста).
- **Стили `<w:pStyle>`** (Heading 1/2/…) → **главы/заголовки** для оглавления и пауз.
- **Метаданные** — `docProps/core.xml` (`dc:title`, `dc:creator`).

**Ловушки DOCX:** текст разбит на множество `<w:r>` (склеивать прогоны абзаца); таблицы
(`<w:tbl>`) — не выдавать кашу; списки; разрывы; колонтитулы (`header*.xml`/`footer*.xml`) —
обычно не для озвучки; сноски `footnotes.xml`.

<a name="txt"></a>
## 6. TXT

Простой текст — кажется тривиальным, но:
- **Кодировка** — UTF-8/UTF-16/windows-1251/CP866; определять (BOM + эвристика), иначе
  кириллица ломается. Это главная ловушка TXT.
- **Переводы строк** — CRLF/LF/CR.
- **Глав обычно нет** — навигация по % книги; не показывать пустой TOC. Можно эвристически
  выделять главы по строкам «Глава N»/заголовкам в верхнем регистре, но осторожно.
- **Очень большие файлы** — грузить/рендерить без блокировки UI.

<a name="общее"></a>
## 7. Общие подсистемы

**Оглавление/главы:** единая модель «книга = список глав (заголовок + текст)». Источник
зависит от формата (spine+nav / FB2-секции / DOCX-стили / PDF-outline). Для подсветки и
навигации храни **символьные смещения** начала глав в плоском тексте.

**Метаданные:** заголовок и автор — из формата; fallback на имя файла. Показывать в
библиотеке и навбаре.

**Обложка:** из формата (EPUB cover-image / FB2 coverpage / PDF первая страница).
Без обложки — генерируемая заглушка с заголовком, а не пустой прямоугольник.

**Модель прогресса (критично, по типу формата):**
- fixed-layout → страница;
- reflow → % книги (+ глава).
Хранить позицию так, чтобы она **переживала** смену размера шрифта (привязка к символьному
смещению/предложению, а не к пикселям/«странице» reflow).

**Производительность и устойчивость:** парс/OCR — off-main и прогрессивно; кэш результата
(с версионированием схемы); понятные ошибки вместо краша на битом/DRM/неизвестном файле.

<a name="чеклист"></a>
## 8. Сводный чеклист обработки формата

При добавлении/ревью поддержки формата проверь:
- [ ] Тип определён: fixed-layout или reflow → выбрана верная модель позиции/прогресса
- [ ] Кодировка определяется корректно (особенно TXT/FB2 — windows-1251!)
- [ ] Порядок чтения верный (EPUB spine, вложенные секции FB2, прогоны DOCX)
- [ ] Извлечены главы/оглавление (если есть в формате)
- [ ] Извлечены метаданные (заголовок/автор) + обложка (или заглушка)
- [ ] Мусор убран (колонтитулы/номера/сноски не лезут в озвучку)
- [ ] Подсветка маппится корректно (бокс для PDF / символьный диапазон для reflow)
- [ ] Тяжёлая работа off-main + прогресс + кэш
- [ ] Битый/DRM/неизвестный файл → понятная ошибка, не краш
- [ ] Позиция чтения переживает смену шрифта (для reflow)
