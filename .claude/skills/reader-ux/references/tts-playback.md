# Конвенции озвучки (TTS) и плеера

Что отличает read-aloud-читалку от обычной: связка «текст ↔ речь» и поведение полноценного
медиаплеера. Ориентиры: Voice Dream Reader, Speechify, @Voice, Pocket Casts, Apple Books
(Speak Screen), Smart AudioBook Player.

<a name="подсветка"></a>
## 1. Подсветка и связь текст↔речь

- **Подсвечивай озвучиваемое** — минимум предложение, лучше слово-в-слово. Это ядро
  read-aloud: пользователь следит глазами и не теряется.
- **Авто-прокрутка** держит активный фрагмент в кадре — но **не у самого края**; держи его
  в комфортной зоне (примерно верхняя треть/центр), чтобы был виден контекст впереди.
- **Точность маппинга.** Подсветка обязана попадать ровно в озвучиваемый фрагмент. Для reflow
  — символьный диапазон в плоском тексте (вербатим-срез → совпадение байт-в-байт); для PDF —
  бокс слова/строки по координатам. Рассинхрон подсветки разрушает доверие.
- **Контраст подсветки** достаточный на всех темах фона (жёлтый по кремовому/сепии часто
  слишком слаб — проверяй на реальном фоне, не на белом).
- **Снятие подсветки** с предыдущего фрагмента при переходе; на паузе подсветка остаётся.

<a name="отсюда"></a>
## 2. «Читать отсюда»

Тап по слову/предложению → начать озвучку с этого места. Конвенции:
- Явное подтверждение (пузырёк/кнопка «Отсюда»), чтобы не путать с иммерсивным тапом/листанием.
- Прыжок по оглавлению/скрабберу/закладке сам по себе **не** запускает речь — только
  переход+подсветка, запуск по явному действию (см. ux-conventions.md, навигация).

<a name="транспорт"></a>
## 3. Транспорт и управление

- **play/pause** — главная кнопка.
- **Шаг назад/вперёд** — определи единицу и подбери иконку под неё:
  - предложение/абзац (типично для read-aloud) → иконки «в край» (`backward.end`/`forward.end`);
  - ±сек (типично для аудиокниг) → круговые `gobackward/goforward` с числом.
  Несоответствие иконки и шага дезориентирует.
- **Переход по главам** — отдельный контрол при наличии структуры.
- Менять скорость/голос **на лету**, не обрывая чтение, где движок позволяет; где не
  позволяет (нативный AVSpeech не меняет темп посреди фразы) — перезапуск текущего фрагмента,
  очередь наполнить заново.

<a name="скорость-голос"></a>
## 4. Скорость и голос

- **Скорость** 0.5×–3× (шагами), быстрый доступ из читалки, значение видно.
- **Голос** выбирается из читалки и из настроек. Если есть разные классы (нейросетевой
  сервер vs системный) — показывай тип/качество, давай превью при выборе.
- **Деградация голоса:** если премиальный/нейроголос недоступен (нет сети) — прозрачно
  использовать системный, не прерывая пользователя (см. офлайн-фолбэк).

<a name="таймер"></a>
## 5. Таймер сна

Ожидаемая фича для прослушивания (особенно художки на ночь).
- Опции: 5/10/15/30/45/60 мин; часто «до конца главы».
- По истечении — **плавно** пауза (в идеале с fade-out), позиция сохранена.
- Индикатор оставшегося времени; лёгкая отмена/продление.
- Размещение: overflow-меню «…» или контрол в плеере; если спрятан в меню — давай намёк,
  что таймер активен (изменённая иконка/бейдж), т.к. обратный отсчёт не виден.

<a name="фон"></a>
## 6. Фон, экран блокировки, перебивки

Раз пошло аудио — это медиаплеер. Обязательные ожидания:
- **Фоновое воспроизведение** — продолжает играть со свёрнутым приложением и на залоченном
  экране (на iOS — `AVAudioSession` категория `.playback`, аудио-режим в Background Modes).
- **Now Playing** — заголовок/автор/обложка/прогресс на экране блокировки
  (`MPNowPlayingInfoCenter`).
- **Remote-команды** — play/pause/next/prev с экрана блокировки, наушников, авто
  (`MPRemoteCommandCenter`).
- **Перебивки** — звонок/другое аудио: корректно встать на паузу и (по политике) возобновить
  после; уважать отключение/подключение наушников.
- **Независимость от экрана** — навигация внутри приложения (уход в библиотеку/настройки)
  **не обрывает** воспроизведение. Непоследовательность «в фоне играет, а при переходе на
  другой экран — пауза» воспринимается как баг.

<a name="офлайн"></a>
## 7. Офлайн-фолбэк

Если синтез зависит от сервера: при недоступности — **беззвучно** перейти на офлайн/системный
голос и продолжить чтение, а не оборвать. Сетевые ошибки не должны быть видимы как сбой.
(Это ещё и снижает риски ревью App Store: офлайн-функциональность сохраняется.)

<a name="позиция"></a>
## 8. Сохранение позиции

- Позиция озвучки/чтения сохраняется постоянно (и при сворачивании, и при закрытии книги) и
  восстанавливается при открытии.
- Хранить устойчиво к смене вёрстки: для reflow — индекс предложения/символьное смещение, не
  «страница»/пиксели. Закладка для read-aloud хранит позицию озвучки.

<a name="нормализация"></a>
## 9. Подготовка текста к синтезу

Качество звучания во многом — это подготовка текста:
- Числа → слова с учётом контекста/склонения; единицы измерения; даты/время.
- Аббревиатуры и сокращения (контекстно: «г.» = год/город/грамм).
- Ударения для омографов (где движок поддерживает разметку ударения).
- Разумные паузы на заголовках/абзацах/пунктуации.
- Чистка мусора (колонтитулы, номера страниц, переносы) — ДО синтеза.
Принцип late render (PDF Voice): хранить сырой очищенный текст, раскрывать в речь при
постановке в очередь — улучшения лингвистики не инвалидируют кэш.

<a name="чеклист"></a>
## 10. Чеклист озвучки

- [ ] Подсветка попадает точно, авто-скролл держит фрагмент в комфортной зоне
- [ ] Контраст подсветки достаточен на всех темах
- [ ] «Читать отсюда» по тапу; навигация не форсит play
- [ ] Иконки переходов соответствуют единице шага
- [ ] Скорость/голос из читалки; превью и тип голоса видны
- [ ] Таймер сна с плавной паузой и индикатором
- [ ] Фон + Now Playing + remote-команды + перебивки
- [ ] Переход между экранами не обрывает звук
- [ ] Офлайн-фолбэк беззвучный, без обрыва
- [ ] Позиция сохраняется и переживает смену шрифта
- [ ] Текст нормализован перед синтезом
