# UX-конвенции читалок

Общепринятые паттерны интерфейса читалок (Apple Books, Kindle, Google Play Books,
Voice Dream Reader, Speechify, Moon+ Reader, FBReader). Цель — пользователь узнаёт
интерфейс по привычке и не учится заново.

<a name="три-экрана"></a>
## 1. Три экрана читалки

Жанр устоялся на трёх уровнях, между которыми пользователь свободно ходит:
- **Библиотека** — список/сетка книг, импорт, обложки, прогресс, «продолжить».
- **Читалка** — сам текст + минимальный хром + плеер.
- **Структура/навигация** — оглавление, закладки, заметки (обычно лист/sheet поверх читалки).

Ключевой принцип: переход между ними **не должен обрывать воспроизведение** (см.
tts-playback.md). Уход в библиотеку выбрать следующую книгу — нормальный сценарий во
время прослушивания.

<a name="библиотека"></a>
## 2. Библиотека

**Что показывает строка/ячейка книги:**
- Обложка (для форматов без обложки — генерируем из первой страницы/заглушка с заголовком).
- Заголовок + автор (если есть в метаданных).
- **Прогресс чтения** — полоска или «N%», и статус «новая / читаю / дочитана». Это важнее
  даты открытия: повторный вход = «где я остановился».
- Маркер «продолжить» для книги в процессе.

**Действия:** импорт (видная CTA, не только «+» в углу), сортировка (недавние/название/автор),
поиск при росте библиотеки, удаление (swipe + контекст-меню), переключение список/сетка.

**Конвенции импорта:** поддержать множественный выбор и системный «Открыть в…/Share»
(пользователь шлёт книгу из почты/браузера). Дубликаты — не плодить молча.

**Чеклист библиотеки:**
- [ ] Видно прогресс и статус каждой книги
- [ ] «Продолжить» очевидно
- [ ] Обложки есть у всех форматов (генерация/заглушка)
- [ ] Импорт заметен и поддерживает несколько файлов / Share
- [ ] Тексты упоминают ВСЕ поддерживаемые форматы, а не только PDF

<a name="экран-чтения"></a>
## 3. Экран чтения и иммерсивный режим

**Анатомия:** контент (страница/текст) занимает максимум; сверху — компактный навбар
(назад, заголовок книги, доступ к оглавлению/закладкам/настройкам); снизу — плеер +
навигация (скраббер/прогресс).

**Иммерсивный режим — почти обязателен.** Один тап по центру текста скрывает/показывает
хром, чтобы читать без отвлечения. Когда хром скрыт — остаётся только текст. (Учесть:
тап по тексту в read-aloud также может означать «читать отсюда» — развести зоны/жесты,
см. Жесты.)

**Заголовок книги** виден в навбаре читалки (ориентация «что я читаю»).

**Состояния, которые экран обязан честно показывать:**
- загрузка/подготовка (парс, OCR) — с прогрессом, а не пустота;
- ошибка извлечения — понятный текст + что делать;
- готово к чтению/озвучке;
- играет/пауза.

**Чеклист экрана чтения:**
- [ ] Иммерсивный режим (тап скрывает хром)
- [ ] Заголовок книги виден
- [ ] Все состояния (загрузка/ошибка/готово/играет) различимы
- [ ] Контент не перекрыт панелями в зоне чтения

<a name="плеер"></a>
## 4. Плеер и транспорт

Раскладка транспорта устоялась — копируем ожидания из аудиоплееров:
- **play/pause** — крупная центральная кнопка, главный акцент.
- **переходы** — назад/вперёд. В read-aloud это «предложение» или «абзац»; в аудиокнигах —
  «±15/30 c». Иконка должна соответствовать действию: круговые стрелки `gobackward/goforward`
  читаются как «секунды», стрелки в край `backward.end/forward.end` — как «пред./след.
  фрагмент». Не вводи в заблуждение.
- **переход по главам** — отдельно от перехода по предложениям, если книга со структурой.
- **скорость** — быстрый доступ (0.5×–3×), типично слева/у края.
- **выбор голоса** — доступен из читалки (не только в общих настройках), особенно если есть
  нейро- и системные голоса.
- **таймер сна** — ожидаемая фича для прослушивания; см. tts-playback.md.

Менее частые действия (таймер сна, голос, тема) разумно собрать в overflow-меню «…» в
навбаре, чтобы не плодить иконки и иметь дом для будущих контролов.

**Чеклист плеера:**
- [ ] play/pause — главный акцент
- [ ] Иконки переходов соответствуют реальному шагу
- [ ] Скорость и голос доступны из читалки
- [ ] Таймер сна достижим
- [ ] Тап-цели ≥ 44pt

<a name="навигация"></a>
## 5. Навигация: оглавление, скраббер, прогресс

**Оглавление (TOC) — обязательно при наличии глав.** Лист со списком глав по заголовкам,
текущая глава выделена, тап → прыжок. Для форматов без структуры (часто TXT) — не показывать
пустой/однострочный TOC.

**Прыжок по навигации не должен принудительно запускать воспроизведение.** Пользователь
часто хочет «посмотреть место». Переход + подсветка, уважая текущее play/pause; запуск — по
явному «читать отсюда».

**Скраббер/прогресс — в единицах, честных для формата:**
- **Fixed-layout (PDF):** «страница N из M», слайдер по страницам, сетка миниатюр.
- **Reflow (EPUB/FB2/TXT/DOCX):** **процент книги** (слайдер по позиции прокрутки/прогрессу)
  + опционально «глава N из M». «Страница N/M» в reflow бессмысленна — текст перетекает.

**Бегунок следует за чтением:** при воспроизведении ползунок/индикатор едет вместе с
подсветкой; перетаскивание — навигация без перезапуска озвучки.

**Закладки:** добавить текущее место одним действием из читалки; список с переходом;
для read-aloud — закладка хранит позицию озвучки, а не только страницу.

**Чеклист навигации:**
- [ ] TOC при наличии глав, текущая глава выделена
- [ ] Прыжок не форсит play
- [ ] Прогресс в правильных единицах (страницы vs %)
- [ ] Бегунок следует за озвучкой
- [ ] Быстрая закладка из читалки

<a name="темы"></a>
## 6. Темы чтения и типографика

**Темы фона** — конвенция жанра: **день (белый) / сепия / ночь (тёмный)**, плюс часто
регулировка яркости. Тема чтения отдельна от системной темы приложения. Для reflow реализуется
полноценно (свой фон/цвет текста); для fixed-layout PDF полная инверсия дорога — реалистичны
тёплый/нейтральный фильтр и яркость.

**Типографика reflow:** регулируемый размер шрифта (и уважение Dynamic Type), межстрочный
интервал, поля, желательно выбор шрифта (в т.ч. для дислексии). Изменения применяются с
сохранением позиции чтения.

**Чеклист тем/типографики:**
- [ ] День/сепия/ночь (или обоснованный минимум)
- [ ] Тема чтения отдельна от темы приложения
- [ ] Размер шрифта regулируется (reflow), Dynamic Type учитывается
- [ ] Смена настроек не теряет позицию

<a name="жесты"></a>
## 7. Жесты

- **Тап по центру** — переключить хром (иммерсивный режим).
- **Тап по слову/предложению** — в read-aloud: «читать отсюда» (с явным подтверждением-
  пузырьком, чтобы не путать с листанием/иммерсивом).
- **Вертикальный скролл** — чтение (в PDF Voice — единственный режим листания).
- **Свайп между книгами/главами** — опционально; не конфликтовать со скроллом.
Разводи конфликтующие жесты по зонам и подтверждениям, иначе тап «спрятать панели»
случайно запускает чтение.

<a name="настройки"></a>
## 8. Настройки

Разделяй **глобальные** (тема приложения, голос по умолчанию, пауза между предложениями) и
**контекстные** (скорость, голос, тема чтения, таймер сна — из читалки). Не заставляй
выходить из книги ради частого действия. Превью голоса при выборе — с явной подсказкой,
показывать тип/качество голоса (нейро/Enhanced/стандартный).

<a name="онбординг"></a>
## 9. Онбординг и пустые состояния

- Короткий онбординг с возможностью **«Пропустить»**; показывает ключевые фичи (импорт всех
  форматов, нейроголоса, закладки, таймер сна), а не только «открой PDF».
- **Пустая библиотека** — не тупик: крупная CTA «Добавить книгу» + перечень поддерживаемых
  форматов.
- Тексты честно перечисляют поддерживаемые форматы.
