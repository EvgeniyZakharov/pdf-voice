# Доступность читалки

Аудитория аудио-чтения непропорционально включает людей со слабым зрением, слепых и
с дислексией — для них read-aloud это не «удобство», а способ доступа к книгам. Поэтому
доступность здесь — часть базовой корректности, а не опция «на потом».

<a name="voiceover"></a>
## 1. VoiceOver

- **Все иконочные кнопки имеют осмысленную метку** (`accessibilityLabel`). Без неё VoiceOver
  читает имя системного символа или молчит. Для каждой кнопки: play/pause, переходы,
  скорость, голос, таймер, закладки, оглавление, миниатюры, импорт, настройки, переключатель
  раскладки.
- **Состояние в метке/value:** play/pause анонсирует текущее состояние; скраббер — `value`
  вида «страница N из M» / «N процентов»; активная глава/закладка различимы.
- **Анонс смены предложения.** При read-aloud полезно сообщать VoiceOver текущий
  озвучиваемый фрагмент (live-region/accessibility-аннотация на области текста) — иначе
  незрячий пользователь не знает, что читается.
- **Логичный порядок обхода** и группировка элементов; декоративное (фоновые оверлеи) скрыто
  от ассистивных технологий (`accessibilityHidden`).
- **Не конфликтуй с жестами VoiceOver:** кастомные тапы («читать отсюда», иммерсивный режим)
  должны иметь доступную альтернативу (custom actions), т.к. одиночный тап перехватывается
  VoiceOver.

<a name="dynamic-type"></a>
## 2. Dynamic Type

- Текст интерфейса масштабируется с системным размером (семантические шрифты / `UIFontMetrics`
  / `ScaledMetric`). Фиксированные `font(size:)` в контролах и тексте reflow — нарушение:
  не растут и обрезаются при крупном размере.
- Текст книги (reflow) уважает пользовательский размер; смена размера сохраняет позицию.
- Проверяй на крупных размерах (вплоть до Accessibility sizes): ничего не обрезается, кнопки
  не наезжают.

<a name="контраст"></a>
## 3. Контраст и цвет

- Текст к фону — достаточный контраст на всех темах (день/сепия/ночь); WCAG AA как ориентир.
- **Подсветка озвучки** различима на всех фонах (полупрозрачный жёлтый по кремовому/сепии
  часто слаб — проверяй на реальном фоне; рассмотри настраиваемый/более контрастный цвет).
- Не кодируй смысл только цветом (активная глава, статус) — дублируй формой/текстом/иконкой.
- Уважай контрастные системные настройки (Increase Contrast).

<a name="тап-цели"></a>
## 4. Тап-цели и моторика

- Интерактивные элементы ≥ **44×44pt** (HIG). Мелкие глифы транспорта/миниатюр оборачивай в
  достаточный `frame`.
- Достаточные промежутки между кнопками, чтобы не промахиваться (особенно в плеере).

<a name="движение"></a>
## 5. Движение и анимация

- Уважай **Reduce Motion**: авто-скролл и переходы — без резких/укачивающих эффектов;
  предлагай мгновенный переход вместо анимации, где включено.
- Авто-прокрутка за подсветкой — плавная и предсказуемая, без рывков.

<a name="чеклист"></a>
## 6. Чеклист доступности

- [ ] Каждая иконочная кнопка имеет `accessibilityLabel`
- [ ] Скраббер/тогглы сообщают `value`/состояние
- [ ] Текущее предложение анонсируется VoiceOver
- [ ] Кастомные жесты имеют доступную альтернативу (custom actions)
- [ ] Dynamic Type работает (UI + текст книги), позиция сохраняется при смене размера
- [ ] Контраст текста и подсветки достаточен на всех темах
- [ ] Смысл не только цветом
- [ ] Тап-цели ≥ 44×44pt
- [ ] Reduce Motion уважается
