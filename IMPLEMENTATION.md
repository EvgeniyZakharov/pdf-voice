## Фаза 0a — швы (поведение озвучки не меняется)

### S1. Late render + модель данных
`Sentence.text → rawText` (+ `language: String = "ru"`). Извлечение хранит СЫРОЕ
очищенное предложение; `expandForSpeech` применяется при постановке в очередь
(playback), а не при извлечении. Кэш: `EncodedSentence` хранит `rawText`+`language`;
схема версионируется → старые кэши (со спикен-текстом) одноразово инвалидируются.
- **Тест:** сборка зелёная; в симуляторе книга открывается, TTS читает те же слова
  (числа/аббревиатуры раскрыты как раньше), подсветка работает; пустые после рендера
  предложения (только-ссылка) не зачитываются; кэш пересоздаётся с `rawText`.

### S2. LanguageProfile + TextPipeline (чистый рефакторинг)
Разнести `TextNormalizer` на: `TextPipeline` (язык-независимое: строки, offset-карта,
склейка переносов, boilerplate, leader/page-number, `mergeCrossPage`) и `RussianProfile`
(токенизация-локаль, `isHeading`, числа, аббревиатуры — реализует протокол
`LanguageProfile`). Поведение идентично.
- **Тест:** Swift-харнесс прогоняет набор страниц через старый и новый путь — массив
  предложений (rawText, ranges, isHeading) побайтно совпадает.

### S3. SpeechBackend
Вынести AVSpeech и Silero из `SpeechEngine` за протокол `SpeechBackend`
(`AVSpeechBackend`, `SileroBackend`). `SpeechEngine` → координатор (очередь-намерение,
подсветка из событий, Now-Playing, прерывания). `render`-замыкание передаётся в бэкенд.
- **Тест:** симулятор — оба движка играют; pause/resume/skip/смена скорости/смена
  голоса работают; фон и экран блокировки не сломаны.

## Фаза 1 — русское качество

### S4. RussianStress
Словарь омографов → позиции ударений в `SpokenMarkup`. `SileroBackend` рендерит `+`
после ударной гласной; `AVSpeechBackend` метки отбрасывает (спайк §9).
- **Тест:** харнесс шлёт на Silero пары (за́мок/замо́к и др.) — WAV различаются; AVSpeech
  не падает и не меняется.

### S5. Согласование числительных
`RussianNumerals`: падеж/род для частых конструкций «предлог + число + сущ.».
- **Тест:** харнесс — таблица вход→ожидаемое («на 2 страницах»→«на двух страницах»).

### S6. Аббревиатуры/единицы
Расширить словарь, разрешать «г.»/«в.»/«см» по контексту.
- **Тест:** харнесс — таблица вход→ожидаемое.

## Фаза 0b — робастность типов PDF

### S7. Постраничный классификатор + единый конвейер
`classifyPage → .text/.ocr/.skip`; OCR и текст идут через ОДНУ очистку
(`TextPipeline`) → паритет (колонтитулы/паузы заголовков на сканах). Смешанные PDF.
- **Тест:** симулятор — текстовый PDF, скан-PDF и смешанный читаются; на скане не
  зачитываются колонтитулы.

### S8. Отдельный OCR-lane + фикс requestPriorityLoad
OCR-страницы — низкоприоритетная очередь, не блокируют текст. Фикс
`requestPriorityLoad` для OCR-документов (`ReaderViewModel.swift:320`).
- **Тест:** симулятор — на смешанном PDF текст доступен сразу, сканы дочитываются
  фоном; прыжок на нераспознанную страницу OCR-книги не даёт немой страницы.
